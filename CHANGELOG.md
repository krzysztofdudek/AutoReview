# Changelog

All notable changes to AutoReview documented here. Format based on [Keep a Changelog](https://keepachangelog.com/).

## [0.2.0]

### Changed (BREAKING)
- **Tiered intelligence redesign — config schema overhauled.** The single-provider `provider:` block (with `active:` selector and per-provider sub-blocks) is replaced by `tiers:`, a map of named tier configs. Five fixed tier names: `default` (mandatory) plus opt-in `trivial`, `standard`, `heavy`, `critical`. Each tier is a self-contained block carrying `provider`, `model`, optional `endpoint`, `parallel`, `mode`, `reasoning_effort`, `consensus`, `timeout_ms`, `context_window_bytes`, `output_max_tokens`. Rules route to a tier via the new frontmatter field. Removed sections: `provider:`, `enforcement:`, `context_overrides:`, `review:`, top-level `rules:`. Removed nested keys: `secrets.ollama.host`. Removed env-var consumption: `OLLAMA_HOST` is no longer read at config-load time (still consumed by `init.mjs` for template generation).
- **Rule frontmatter: per-rule `provider:` / `model:` / `intent:` / `evaluate:` / `default: disabled` removed.** New fields: `tier:` (one of the five named tiers; defaults to `default`), `severity:` (`error` or `warning`; defaults to `error`), `type:` (`auto` or `manual`; defaults to `auto`). `description:` stays unchanged. Soft-fail validation: rules with unknown values for `tier` / `severity` / `type` register but are marked `_invalid` and produce `[error]` verdicts on every match (one bad rule no longer blocks the rest of the load).
- **Pre-commit hook exit-code policy is now per-rule severity-driven.** A `severity: error` rule producing `[reject]` OR `[error]` (provider unreachable, network, timeout, truncation, undefined tier) blocks the commit (exit 1). A `severity: warning` rule never blocks (prints as `[warn]` for rule violations, `[error]` for tooling failures, exit 0). The previous CLAUDE.md hard rule "pre-commit hooks must never hard-block on provider errors" is removed; severity is the opt-out. The `enforcement: precommit/validate` config knob and the corresponding `--context`-driven soft/hard switch are gone.
- **Remote-rule overlays.** New `remote_rules[].overrides[<rule-id>]` block in config lets a repo adapt an upstream rule's frontmatter without forking. Overridable fields: `name`, `triggers`, `tier`, `severity`, `type`, `description`. Body is never overridable. Null-as-unset semantics: `tier: null` reverts to default. Personal config overlays merge over repo overlays by `name`. Unknown override field name throws at config-load; invalid values follow the soft-fail path described above.
- **CLI surface narrowed on `validate.mjs`** — `--mode` and `--reasoning-effort` flags dropped (mode and reasoning effort are tier-level config). The `--context` flag now only affects scope default (`precommit` → staged, `validate` → uncommitted) and the precommit-only `[hint]` block; it no longer carries an enforcement policy. New flag: `--rule <id>` is mandatory when invoking `type: manual` rules; non-`manual` rules also accept it for narrowing. `--rule` with non-matching triggers on the supplied files prints a `[warn]` line ("rule X declared via --rule but triggers don't match Y; skipping") and skips — was previously a silent no-op.
- **Legacy config keys are rejected at load time with friendly migration messages.** `provider:`, `enforcement:`, `context_overrides:`, `review:`, top-level `rules:` (legacy `disabled` / `enabled_extra` lists), and `secrets.ollama.host` each throw a per-key error pointing at the new equivalent. Previously the loader silently ignored unknown keys, which masked stale configs after the schema migration.
- **Layer 2 NL intent-gate feature removed.** The `intent:` rule frontmatter field, the `review.intent_triggers` and `review.intent_trigger_budget` config knobs, and the `scripts/lib/intent-gate.mjs` module are deleted entirely. Existing rules with `intent:` in frontmatter pass it as an unknown field which is silently ignored (the field is no longer read by any code path).
- **Slash commands removed; skills are the only Claude Code surface.** All 8 entries under `commands/` (`init`, `validate`, `check-breadth`, `history`, `pull-remote`, `context`, `guide`, `create-rule`) deleted. Their unique content was migrated into the matching skill body (or a new skill — `history` and `pull-remote` are now skills too, bringing the total to 8). Rationale: every command was either a thin doc-redirect to a skill or a deterministic CLI shortcut that the agent can invoke from a skill's body via `node ${CLAUDE_PLUGIN_ROOT}/scripts/bin/<X>.mjs`. Users typically never typed slash commands directly — they asked the agent in natural language and the agent picked a skill anyway. Eliminating the parallel command surface removes slug collisions in the slash menu and unifies the agent's decision path. Migration table:
  - `/autoreview:init` → `autoreview:setup` skill
  - `/autoreview:validate` → `autoreview:review` skill
  - `/autoreview:check-breadth` → step 3 of the `autoreview:create-rule` wizard
  - `/autoreview:history` → `autoreview:history` skill (new)
  - `/autoreview:pull-remote` → `autoreview:pull-remote` skill (new)
  - `/autoreview:context`, `/autoreview:guide`, `/autoreview:create-rule` → same-named skills (existing)
  Pre-commit hook is unaffected — it always invoked `validate.mjs` directly via shell, never via a slash command.

- **Skill directories renamed: `skills/autoreview-<name>/` → `skills/<name>/`.** The redundant `autoreview-` prefix produced ugly double-namespacing (`autoreview:autoreview-create-rule`) in the system prompt's available-skills list and inconsistent slash-menu styling between commands and skills. After rename every skill is `autoreview:<name>` matching the convention from `superpowers:writing-skills`. Affected: `setup`, `create-rule`, `review`, `context`, `guide`, `precheck`. Cross-references in templates, README, CLAUDE.md, and docs updated.

### Changed
- **`type: manual` rules skip silently unless explicitly invoked via `--rule <id>`.** Auto-rules continue to run on every `validate` / pre-commit pass per their triggers; manual rules now require explicit opt-in. `autoreview:context <path>` lists manual rules with a `[manual]` marker so authors can see them. Soft-failed rules (per the new `_invalid` path) are listed with `[invalid: <reason>]`.
- **Per-tier semaphore for concurrency limiting** — `parallel` is now keyed by tier name in `provider-client.getProvider({ tierName })`. Two tiers using the same provider account get separate semaphores. Same-account tiers no longer share a single rate-limit pool; the existing 429/503 retry-with-backoff handles transient overlap. Cache key for the wrapped provider also includes the tier name, so renaming a tier flushes the cache cleanly.
- **`[hint]` block on a precommit reject prints only when at least one failed rule used `mode: quick`.** Failures from `mode: thinking` tiers don't get the hint because their reason is already printed inline by `report.mjs`. Hint wording rewritten to point at editing `tiers.<name>.mode` in `.autoreview/config.yaml` instead of the removed `--mode thinking` flag.
- **Verdict prefixes are severity-aware.** `report.mjs` now selects the user-facing prefix based on per-rule severity: `severity: error` violations stay `[reject]` (preserved to avoid churning snapshots and hook parsers); `severity: warning` violations print as `[warn]`. Provider/tooling failures print as `[error]` regardless of severity. The internal verdict label (`fail` / `error` / `pass` / `suppressed`) is unchanged in `history.jsonl` records — only the printed prefix flips.
- **Verdict log records carry `tier` and `severity` fields.** Existing `.autoreview/.history/*.jsonl` records from prior versions are wiped at install; new records carry both. The `history` CLI gains `--tier <name>` and `--severity <error|warning>` filters.
- **Committed default provider is `ollama`** (was `openai-compat`). The committed `.autoreview/config.yaml` is what every fresh clone — including the opensource newcomer — picks up first; pointing it at a generic local LLM matches the project's "Ollama-first, offline" framing better than naming a specific cloud-compat surface. Per-machine endpoints (`llama-server`, MLX, paid APIs) belong in the gitignored `config.personal.yaml`, which deep-merges over the committed config. The `openai-compat` block remains in `config.yaml` as a placeholder (`http://127.0.0.1:8080/v1`, `local-model`).
- `agent-rules.md` soft token budget bumped 800 → 1000 (hard ceiling stays at 1200). The cross-platform notes added to skill bodies in this release pushed `templates/agent-rules.md` to ~941 rough tokens — within budget but past the old soft warning. Real ceiling unchanged; only the noise-level alarm moved.
- npm test scripts restructured for clearer separation: `test` = unit (lib + bin + plugin + api), `test:e2e` = e2e against an OpenAI-compat server, `test:ollama` = real Ollama round-trip, `test:all` = unit + e2e, `coverage` = 90% gate. The old `test:lib` script (a mis-globbed subset that silently skipped `tests/lib/providers/`) was dropped.
- `bin-shape` rule (`.autoreview/rules/bin-shape.md`) updated to enforce the `isMainModule(import.meta.url)` invocation guard introduced in 0.1.1 (cross-platform fix), and to explicitly forbid the legacy `import.meta.url === \`file://${process.argv[1]}\`` form. Previously the rule still required the legacy form, so every `scripts/bin/*.mjs` file added or touched in 0.2.0 produced a stale-rule `[reject]` until the rule was rewritten.

### Added
- **New `autoreview:override-rule` skill (9th skill total)** — wizard for adapting remote-rule frontmatter via `remote_rules[].overrides`. Backing CLI: `scripts/bin/override-rule.mjs save --remote <name> --rule <id> --field key=value [--scope repo|personal]`. Performs atomic YAML edits via `writeAtomic`, preserves comments and existing structure, supports `--field tier=null` for null-as-unset semantics.
- **`override-rule save` validates `--remote` / `--rule` / `--field` arguments** for safe YAML output. `--remote` must match `[A-Za-z0-9._-]+`, `--rule` must match `[A-Za-z0-9_/-]+`, field values must not contain newlines. Prevents YAML injection from agent-supplied args (e.g. a newline in `--remote` injecting a sibling `url:` key into `config.yaml`).
- **Tier-resolution failures become per-rule `[error]` verdicts.** Rule with `tier: critical` against a config that defines only `default` produces an `[error]` verdict for that rule with reason `tier 'critical' not defined in tiers: in .autoreview/config.yaml`. Other rules in the same file continue. Severity governs whether this contributes to exit 1 (default `severity: error` does block; `severity: warning` doesn't).
- **Per-provider `timeout_ms` config field.** Every provider block (ollama, openai, openai-compat, anthropic, google, claude-code, codex, gemini-cli) now accepts an optional `timeout_ms` (positive integer) that caps a single `verify(prompt)` call. Default stays `120_000`. Plumbed through `provider-client` → factory → HTTP request `timeoutMs` (network providers) or `runCli` `timeoutMs` (CLI providers). Crucial for local LLMs in thinking mode where one call can run 5+ minutes against a full file + diff prompt.
- **`claude-code` provider runs the CLI in maximum-isolation mode** to keep the caller's environment out of the verdict prompt. New flags: `--tools ""` (no built-in tools), `--disable-slash-commands` (no skills loaded), `--setting-sources ""` (no user/project/local settings — also kills any SessionStart hook injections), `--strict-mcp-config --mcp-config '{"mcpServers":{}}'` (no MCP servers), `--no-session-persistence`, `--exclude-dynamic-system-prompt-sections` (cwd/env/git-status out of system prompt), `--output-format json` (structured envelope). Result: the only context the model sees is the AutoReview prompt itself — no `CLAUDE.md` from the user's repo, no skills from `~/.claude`, no tools to call. Wall-clock per call drops from ~5s to ~3s for trivial prompts; cost drops in proportion. The provider parses the `--output-format json` envelope, extracts `result`, feeds it to `parseResponse`, and surfaces `usage.{input_tokens, output_tokens, total_tokens}` to the consensus aggregator.
- Two new skills: `autoreview:history` (query verdict log; replaces the deleted `history` command with a richer "Use when…" trigger description for agent discovery) and `autoreview:pull-remote` (fetch remote rule sources; same).
- Cross-platform note in every skill body explaining `${CLAUDE_PLUGIN_ROOT}` / `$env:CLAUDE_PLUGIN_ROOT` / `%CLAUDE_PLUGIN_ROOT%` translation between bash, PowerShell, and cmd. Skills now tell the agent to detect platform with `node -e "console.log(process.platform)"` when shell-syntax matters.
- `setup` skill: Ollama probe rewritten from `curl -sf` to `node -e "fetch(...)"` — works on every platform (Windows without git-bash, Linux, macOS) since Node ≥22 is the only assumed binary. Same swap for the model-list step.
- `precheck` skill: tmp file path resolved via `node -e "console.log(require('os').tmpdir())"` instead of hard-coded `/tmp/draft-<random>.ts` — works on Windows where `/tmp` doesn't exist by default.
- `.devcontainer/` for in-container dogfooding. Boots Node 22 (TypeScript image, symmetric with the Yggdrasil family), installs `@anthropic-ai/claude-code` globally, registers this repo as a local Claude Code marketplace, and installs `autoreview@autoreview-marketplace` at user scope so slash commands and skills work out of the box. Targets a host-side `llama-server` on port `8089` via `host.docker.internal` for the `openai-compat` provider.
- `.env.example` + per-developer `.env` (gitignored) for test configuration. `npm run test:e2e` and `npm run coverage` load it via Node 22's `--env-file-if-exists`, so devs override `AUTOREVIEW_E2E_ENDPOINT` / `AUTOREVIEW_E2E_MODEL` once instead of retyping the prefix every run. No new dependency — Node stdlib.
- New npm scripts: `test:ollama` (round-trip against a real Ollama daemon, gated by `AUTOREVIEW_REAL_OLLAMA=1`) and `test:all` (unit + e2e in one command). Both documented in CLAUDE.md, CONTRIBUTING.md, and README.

### Fixed
- **`scripts/bin/autoreview.mjs` dispatcher now wraps `run()` in an outer `try/catch`.** A throw from `await loader()` (dynamic import failure, e.g. typo in `SUBCOMMANDS` map or a renamed file) used to surface as an uncaught rejection with raw stack trace; now writes `[error] internal: <stack>` to `ctx.stderr` and returns `2`, matching the `bin-shape` rule contract that every other `scripts/bin/*.mjs` already followed.
- **Trigger evaluation silently produced zero matches on Windows** for any `dir:` or `path:`-anchored expression. `walk()` yields native paths via `path.join()` (`\` separator on Windows); `trigger-engine.evaluate` then matched them against globs whose regex anchors used `/`. Path predicates that didn't anchor on a directory (`**/*.cs`) accidentally worked because the optional `(?:.*/)?` prefix let the trailing `[^/]*\.ext` consume the entire backslash path. Anything more specific returned 0 matches. `evaluate()` now normalises `ctx.path` with `split(sep).join('/')` before regex test, matching the existing idiom used by `gitignoreMatch` in `fs-utils.mjs`.
- **Rule ids contained literal backslash on Windows.** `rule-loader.loadOne` built ids from `relative(localDir, file)` without normalising separators, producing ids like `error-handling\foo` that broke `--rule` lookups, JSON output, and any cross-platform text reference. Now normalised the same way as trigger paths.
- **`saveRule` rejected every save on Windows** with `relativePath escapes .autoreview/rules`. The boundary check used a hard-coded `'/'` separator (`!abs.startsWith(rulesDir + '/')`); both `rulesDir` and `abs` come from `resolve()` and use `\` on Windows, so the prefix never matched. Now uses `path.sep`. Same fix applied to the parallel boundary check in `context.mjs:relTarget` and `scope-resolver.mjs:rel`.
- **`pullSource` rejected file:// remotes whose URL was a Windows-absolute path** (`C:\Users\…\repo`). The validator regex only accepted POSIX-absolute paths (`/...`). Test fixtures using `mkdtemp` for fake remotes failed on Windows. Regex extended to accept `[A-Za-z]:[\\/]` as a prefix and backslash in the body. `git clone` is invoked via `spawn(['git', 'clone', '--', url, …])` so backslash inclusion is shell-injection safe.
- **`scope-resolver` and `check-breadth` displayed paths with native separators** (`Source\api\foo.ts` on Windows) inconsistent with how triggers and rule ids are spelled. Both now normalise printed/pushed paths to forward slashes.
- **`saveRule` returned a Windows-native absolute path** (`C:\Users\…\rules\foo.md`) where tests asserted `.endsWith('.autoreview/rules/foo.md')`. Returned value now normalised to forward slashes for consistent display + test assertions across platforms.
- **`runCli` test fixtures used `.sh` scripts** which `child_process.spawn` cannot execute on Windows (`EFTYPE` — kernel doesn't know how to interpret a shebang). Replaced with `.mjs` fixtures invoked via `process.execPath` — cross-platform, no shell dependency.
- Test-side platform-awareness in `tests/lib/fs-utils.test.mjs` (walk + gitignore + pluginRoot), `tests/lib/git-utils.test.mjs` (repoRoot): assertions now normalise actual path with `posix(p) = p.split(sep).join('/')` before string-matching against `/`-anchored expectations.
- `hooks/hooks.json` used a flat `{matcher, type, command}` form for `SessionStart` entries, which Claude Code ≥2.x rejects with `Hook load failed: expected array, received undefined` at path `hooks.SessionStart.0.hooks`. The plugin installed cleanly but loaded with `Status: ✘ failed to load`, so the SessionStart runtime-sync never fired and `.autoreview/runtime/` stayed pinned to whatever version was first written. Wrapped the command in the nested `{matcher, hooks: [{type, command}]}` shape now required by Claude Code. The `sessionstart-hook-shape` rule and `tests/plugin/hooks.test.mjs` were updated in lockstep.
- `tests/e2e/real-ollama.test.mjs` failed loud with `[error] != [pass]` when the Ollama daemon was unreachable, instead of skipping like every other e2e test. Added an `OLLAMA_HOST/api/tags` HEAD probe; the test now skips with `Ollama daemon http://localhost:11434 unreachable` when the daemon is down — matching the `serverAvailable()` pattern used by the rest of the e2e suite.
- `npm run test:lib` glob (`tests/lib/*.test.mjs`) silently missed `tests/lib/providers/` — the 8 provider test files only ever ran via the broader `npm test` glob. Old `test:e2e` glob (`tests/e2e/cli-*.test.mjs`) similarly skipped `smoke.test.mjs` and `real-ollama.test.mjs`. The replacement scripts use `tests/lib/**/*.test.mjs` and explicit `tests/e2e/cli-*.test.mjs tests/e2e/smoke.test.mjs` lists, catching every intended file.
- **Semaphore queue-timeout bug**: `consensus.voteConsensus` wrapped `provider.verify(prompt)` in a `Promise.race` against a hardcoded 120s timer. The race timer started ticking the moment the verify *promise* was created — i.e. *before* `sem.acquire()` in the wrapped provider — so under fan-out (`Promise.all` over many `(file, rule)` pairs with `parallel:1`), every queued call's clock was already running. Calls whose queue position exceeded 120s died in the queue with the LLM at 0% load, all reporting `duration_ms ≈ 120000` simultaneously. Effect: a 102-file `--sha` review of release 0.2.0 produced 254 timeout errors in 6 minutes wall-clock against a sequential 21-hour budget. Removed the consensus-level race entirely; each provider already enforces its own timeout (HTTP `timeoutMs` for network providers, `runCli` `timeoutMs` for CLI), and that timer correctly starts when the slot is held. Regression test added in `tests/lib/concurrency.test.mjs`.
- `git-utils.commitFiles(sha)` returned every path touched by the commit including deletions, so callers reading content via `fileAtCommit(sha, path)` exploded with `fatal: path 'deleted/file.md' does not exist in <sha>` — the entire `validate --sha <commit>` flow refused to start whenever the commit had any deletes. Switched to `git show --name-status` and skip `D` entries; rename/copy entries return the destination path. Test added.
- `scripts/bin/validate.mjs` reached for `appendFile` via `await import('node:fs/promises')` even though `readFile` was already statically imported from the same module on line 14. Folded `appendFile` into the existing static import — the `dynamic-import-justified` rule caught this in self-review of 0.2.0.

### Performance
- None.

### Added (per-provider parallelism)
- **Per-provider parallelism cap**: each provider config block now accepts an optional `parallel` field (positive integer) that caps concurrent in-flight LLM calls to that provider. Worker-pool semantics — when a slot frees, the next queued task starts. Defaults: `ollama: 1`, `openai-compat: 5`, `anthropic`/`openai`/`google: 10`, `claude-code`/`codex`/`gemini-cli: 3`. Backwards compatible (no field → default applied). See README "Providers" section.
- HTTP `429`/`408` responses are now retried with exponential backoff. `Retry-After` headers (delta-seconds and HTTP-date forms) are honoured, capped at 30s. Each retry surfaces a single `[warn]` line on stderr.
- `validate.mjs` fans out reviews across `(file, rule)` pairs via `Promise.all`. Verdict lines stream eagerly in completion order; the pre-existing `[pass]/[reject]/[error]/[suppressed]` line format is unchanged.

### Changed (per-provider parallelism)
- `withRetry` default `attempts` bumped from 3 to 4.
- `file-summary` history records are now emitted by `validate.mjs` (per-file aggregator), not by `reviewer.mjs`. Format unchanged.

## [0.1.1]

### Fixed
- `chunker.fitFile` produced a negative `sliceBytes` when the available budget was ≤ the 16-byte truncation marker, yielding truncated output that silently exceeded the context window. Guard added — sub-marker budgets now `skip` with a clear reason.
- `history.truncateFileField` crashed with `TypeError: Buffer.from(undefined)` whenever an oversize record (>3500B) lacked a `file` field (e.g. provider-error records carrying a large `raw`). Missing `file` is now a no-op; `raw` gets a final truncation pass so the line stays within `MAX_RECORD_BYTES`.
- `args.parseArgs` silently accepted `--flag` without a value, writing `undefined` into the config, and happily swallowed the next flag as the value (`--mode --rule foo` → `mode='--rule'`). Missing values now throw an explicit `--flag requires a value` error.
- `trigger-engine.toRegex` infinite-looped on globs with an unterminated `[` (e.g. `[abc` with no closing `]`): `indexOf(']', i)` returned `-1`, the for-loop's `i++` reset `i` to `0`, re-processing the same input forever and hanging the reviewer. Now throws `unterminated '[' bracket in glob`.
- Every `scripts/bin/*.mjs` entrypoint silently no-opped on Windows. The main-module guard `import.meta.url === \`file://${process.argv[1]}\`` never matched — `import.meta.url` is `file:///C:/...` while the constructed URL used backslashes and two slashes. Node imported the module, top-level code ran, but `run()` was never invoked. Exit 0, no output, no artifacts. First-time Windows users saw `init --install-precommit` "succeed" with no `.autoreview/` and no hook installed. Replaced with a cross-platform `isMainModule()` helper in `fs-utils.mjs` using `pathToFileURL(argv[1]).href`. Applied to all 11 entrypoints.

### Changed
- **Node ≥22 required.** The `npm test` glob pattern `tests/**/*.test.mjs` is expanded by `node --test` itself; Node 20 does not support `**` as a test-file pattern and every CI run exited with "Could not find tests/**/*.test.mjs". Node 20 hit end-of-life in April 2026, so `engines.node` is bumped rather than worked around. CI matrix updated to `[22.x, 24.x]`.

### Fixed
- `runCli` in `cli-base.mjs` threw an uncaught `EPIPE` on fast kernels (Linux/CI) whenever the child process closed its stdin pipe before the parent finished writing. The Claude Code and Codex provider tests were flaking on GitHub Actions with `failureType: uncaughtException, error: 'write EPIPE'`. A `child.stdin.on('error', () => {})` handler now swallows the write-side close — the real outcome is already captured via stdout/close events.

### Added
- Automatic `.autoreview/runtime/` upgrade on SessionStart. Previously, bumping the plugin left the bundled runtime pinned to whatever version was installed the last time `init --upgrade` ran, so pre-commit hooks kept executing stale code indefinitely. A `.autoreview/runtime/.version` sentinel is now written on init; SessionStart compares it against the installed plugin manifest and re-copies `scripts/lib/` + `scripts/bin/validate.mjs` when they diverge. `[autoreview] runtime upgraded X → Y` reported to stdout so the agent can relay the change. Shell / CI invocations (no `CLAUDE_PLUGIN_ROOT`) are unaffected — they keep using the bundled copy as before.

### Changed
- `init` no longer modifies the user's repository root `.gitignore`. A dedicated `.autoreview/.gitignore` is written instead (git honors nested per-directory `.gitignore` files), so AutoReview's runtime artifacts are ignored without touching any of the user's existing patterns.
- README, plugin manifest, and marketplace description repositioned away from "LLM reviewer" as the lead framing toward "per-file architecture gates" / "rule engine". Mechanism description unchanged deeper in the README. Keywords dropped `code-review`, `llm`, `linter`, `convention-enforcement`; added `architecture-enforcement`, `rule-engine`, `agentic-guardrails`, `agents-md`, `claude-md`, `markdown-rules`, `pre-commit-hook`. FAQ gained "Is this just another AI code review bot?". Footer "Yggdrasil family" link now clickable.

## [0.1.0]

### Added
- Initial implementation covering all 29 points of the functional specification.
- Claude Code plugin with 6 skills, 8 slash commands (including `/autoreview:history`), and SessionStart hook.
- Zero-dep Node stdlib implementation. Runs on Node 20+.
- 225 tests, 1 env-gated smoke test for real Ollama.
- 8 provider adapters: Ollama, Anthropic, OpenAI, Google, OpenAI-compatible, Claude Code CLI, Codex CLI, Gemini CLI.
- Public JS API via `scripts/index.mjs` and unified CLI via `scripts/bin/autoreview.mjs`.
- `review.output_max_tokens` config (default `0` = no limit) to cap LLM output on paid APIs or let local models finish their thinking trace.
- E2E test suite (~140 scenarios) covering every CLI + a 90% coverage gate (`npm run coverage`) on lines, branches, functions.

### Fixed
- Response parser mis-picked the input file's JSON when a model quoted it in its reasoning trace. Now scans all candidates and takes the last one that carries a `satisfied` key.
- `scope-resolver` silently dropped content for `--files <absolute-path>` via `path.join(repoRoot, absPath)` producing a bogus path. Now detects absolute paths and normalizes to repo-relative.
- `validate` + `reviewer-test` `--content-file` resolved against `process.cwd()` instead of the CLI's `ctx.cwd`.
- `create-rule --body-file` same issue — relative paths from Claude Code's cwd now resolve correctly.
- `saveRule` now guards against path traversal (`../`, absolute paths) on `--to`.
- `worktreeModifiedPaths` includes untracked files so `--scope uncommitted` matches the name.
- `remote-rules-pull` preserved `path:` layout on `cp` so rule-loader's base matches the pulled tree.
- Thinking-mode suppress + reasoning warnings go through `ctx.stderr` instead of `console.error`.
- Prompt no longer requests `reason` on `satisfied=true`; parser drops it client-side too.
- Quick-mode `maxTokens: 100` hardcode was too tight for reasoning-first models (Qwen, R1, o1) — their short reasoning trace exhausted the cap before the verdict JSON, producing silent `[error]` verdicts that let rule violations slip past hard enforcement. Unified both modes under a single `review.output_max_tokens` knob with default 0 = no cap. Adapters handle 0 per their API (omit field / `num_predict: -1` / 8192 fallback for Anthropic).
- Truncated-file verdict no longer silently passes. When a file exceeds the context window but fits within 3× (chunker's truncate branch), a `satisfied: true` verdict on partial content is now promoted to `[error] truncated: reviewer saw only first N bytes of M — pass verdict on partial content is unreliable`. `satisfied: false` still counts as a real reject (violation found in what the reviewer saw).
- README `autoreview validate …` shell example replaced with real invocations: `/autoreview:validate …` (Claude Code) + `node .autoreview/runtime/bin/validate.mjs …` (shell). The bare `autoreview` binary requires `npm install -g` which plugin-marketplace users don't do.
- `[reject]` hint block `${AUTOREVIEW}` placeholder replaced with the real post-init path `node .autoreview/runtime/bin/validate.mjs`.

### Changed
- Pre-commit hook template passes `"$@"` through to the CLI.
- `init --upgrade` + `--precommit-{skip,append}` branches covered by tests.
- `[reject]` lines under soft enforcement are now tagged `(warn-only — commit proceeds under soft enforcement)` so users don't mistake a soft-mode warning for a hard block.
- `[reject]` hints now include `why (Claude Code)` + `why (shell)` + `skip:` + `edit:` + `help:` lines inline. Blocked users don't have to leave the terminal to know what to do.
